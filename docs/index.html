There’s a solution: Open Science

“Open Science is an umbrella term encompassing a multitude of assumptions about the future of knowledge creation and dissemination.” (Fecher & Friesike, 2014)

At a high-level, it can be defined as the following.

“Open Science is scholarly research that is collaborative, transparent and reproducible and whose outputs are publicly available.” (European Commission & Directorate-General for Research and Innovation, 2018)

What is data after all?

Data is an abstraction of reality. It’s a representation of the world around us. Without context, it has no meaning.

Data engineering

“Data engineering is the development, implementation, and maintenance of systems and processes that take in raw data and produce high-quality, consistent information that supports downstream use cases, such as analysis and machine learning.”

You can think data engineering as the plumbing of data science.

(Reis & Housley, 2022)

Data science

It’s a confusing term, with many definitions and interpretations.

For some, data science is just statistics (Broman, 2013) (hype statistics). For others, it’s a new interdisciplinary field that synthesizes statistics, informatics, computing, communication, management, and sociology (Cao, 2017).

The arguments about the difference between data science and statistics are not very good. So it’s best to use a high-level definition:

“Data science is the study of the generalizable extraction of knowledge from data.” (Dhar, 2023)

(Figure extracted from Wickham et al. (2023))

Data engineering versus data science

“Data scientists, according to interviews and expert estimates, spend from 50 percent to 80 percent of their time mired in this more mundane labor of collecting and preparing unruly digital data, before it can be explored for useful nuggets.” (Lohr, 2014).

Open licenses

A unlicensed work is an “all rights reserved” work. This means that you can’t use it without the author’s permission.

  • Licences for data
    • Open Knowledge Foundation
  • Licences for software/code
    • OSI Approved Licenses
    • The MIT licence
    • GNU General Public License version 3
  • Licences for everything else (e.g., documents, figures)
    • Creative Commons
    • CC BY 4.0

Some other things that we didn’t cover

We just scratched the surface of the open science world. There are many other tools and concepts that we didn’t cover, such as:

  • Pipelines (e.g., targets).
  • Reproducible environments (e.g., renv, Docker).
  • Markup languages (e.g., Markdown, LaTeX).
  • Open source softwares (e.g., rOpenSci, CRAN, Bioconductor, PyPI).
  • Version control systems (e.g. Git, SVN, Semantic versioning).
  • Open source data management systems (e.g., CKAN, Dataverse).
  • Open journals (e.g., PLoS, BMJ Open, PeerJ).
  • Community-driven open journals (e.g., Journal of Statistical Software, JASSS).
  • Preprint servers (e.g., arXiv, bioRxiv, OSF preprints).
  • Git-based repositories (e.g., GitHub, GitLab).
  • Open repositories (e.g., Zenodo, FigShare, ResearchGate, Dryad).
  • Data repositories (e.g., GSA, UniProtKB, OpenNeuro).
  • Open science policies (e.g., FAPESP, European Commission).
  • Shadow libraries (e.g., Sci-Hub, Library Genesis).

Suggestions for the Global Syndemic project

  • Start using a project management tool (e.g., OSF).
  • Start using a data management system.
  • Start using dynamic documents for the data analyses.

Two main things

  • Centralize all data in a single place. (OSF)
  • Tidy and document all data.
